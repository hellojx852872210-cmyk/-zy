# CLAUDE.md

## 线上变更零中断 SOP（强制）

目标：任何修改都不得影响现有可用功能。

1. 只读排查优先
- 先定位根因与影响面，不直接改线上运行文件。

2. 变更前声明
- 执行前必须先说明：改动文件、改动点、预期影响、回滚方式。

3. 最小补丁原则
- 一次只改一个问题，不做顺手重构，不混入无关改动。

4. 构建/语法闸门（必须通过）
- 前端改动后必须先 `npm run build` 成功。
- 任何报错或告警异常先处理，不允许带病重启。

5. 路径一致性检查（前端必做）
- 子路径部署必须检查 Vite `base` 与 Nginx 路由一致。
- 发布后检查 HTML 中 assets 路径是否与部署路径一致。

6. 分步重启
- 每次只重启一个服务（web/api），观察状态为 online 再继续。

7. 发布后探活（必须通过）
- 页面入口返回 200。
- 关键静态资源返回 200。
- 关键 API 返回 200（至少健康检查+一个业务接口）。

8. 失败即回退
- 任一步失败立即回退到上一个可用版本，不继续在故障态试错。

9. 结果回执模板
- 仅汇报三项：改了什么、如何验证、如何回滚。

## 本项目当前部署约束
- 看板挂载路径：`/repair-dashboard/`
- 前端构建 base 默认应为：`/repair-dashboard/`
- 本地 API 入口：`/local-api/*`（经 Nginx 代理）

## 执行承诺
- 后续所有线上修改严格按本文件执行；若任一步无法满足，先停手并向用户确认。
